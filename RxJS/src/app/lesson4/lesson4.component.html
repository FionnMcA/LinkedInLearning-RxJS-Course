Operators

RxJS is mostly useful for its operators, even though the Observable is the foundation. Operators are the essential pieces that allow complex asynchronous code to be easily composed in a declarative manner.

So so far we had only been able to transform data once something subscribes to an observable and data is passd to it. But sometimes we want to transform the data before a subscription gets to it and after an observable gets it
So in this example we round down the temperature to the nearest whole number
Operators in RxJS are mainly ised to transform data paths to an observable. The changes data will eventually reach the subscription function

  • Only transforming data? Use an operator
  • Finished and need to interact with the component? write it in the subscription

In this example we used pipe and map
SO what do these do?
Pipe: is used to combine multiple operators into a single operation chain
Map: Here the map operator is transforming the weather object emitted by the weatherSubject$. Specifically it rounds up the temperature and returns a new weather object with the transformed temperature and original day
