When we have our weather app we have a button thats clicked and emits and event to an observable passed to the subscribers. If someone clicks this multiple times it can be costly (performing multiple unneccessary api calls). We need to make sure we're carefull about a user clicking too often
A way to do this is to wait until the previous event has been fired before creating a new one. We do this using exhaustMap.
An Exhaust - An RxJS operator that waits for the previous event to finish before starting a new way
